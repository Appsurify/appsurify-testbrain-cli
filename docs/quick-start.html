Appsurify Testbrain CLI Help

Quick start

Welcome to the Appsurify Testbrain Command-Line Interface (CLI)! This article introduces the CLI and helps you complete common tasks.

Installation

pip3 install appsurify-testbrain-cli
poetry add appsurify-testbrain-cli
python3 -m pip install appsurify-testbrain-cli
    docker pull appsurifyinc/appsurify-testbrain-cli:latest
    docker pull appsurifyinc/appsurify-testbrain-cli:edge

      Get Token (API KEY)

      testbrain auth token \ --server [server] \ --email [email] \ --password [password]
      tbcli auth token \ --server [server] \ --email [email] \ --password [password]

              Run push changes GitHub Actions

              1. On the Settings tab of the GitHub UI, create the TESTBRAIN_TOKEN encrypted secret and save the testbrain server token as its value.

              2. On the Settings tab of the GitHub UI, create the variables and save.

                • TESTBRAIN_SERVER - Testbrain server url (ex. https://demo.appsurify.com)

                • TESTBRAIN_PROJECT - Project name on Testbrain server.

              3. On the Actions tab of the GitHub UI, set up a new workflow and create the .github/workflows/testbrain.yml

              4. To inspect the main branch, release branches, and the pull requests coming to your repository, save this workflow configuration to the .github/workflows/testbrain.yml file:

                name: "Testbrain" on: workflow_dispatch: push: branches: - "main" jobs: delivery: name: "Delivery" runs-on: ubuntu-latest steps: - name: "Checkout git" uses: actions/checkout@v4 with: fetch-depth: 0 - name: "Set up Python" uses: actions/setup-python@v4 with: python-version: 3.12 - name: "Set up Testbrain CLI" run: pip3 install appsurify-testbrain-cli - name: "Delivering changes to the server" run: | testbrain repository git push \ --server ${{ vars.TESTBRAIN_SERVER }} \ --token ${{ secrets.TESTBRAIN_TOKEN }} \ --project ${{ vars.TESTBRAIN_PROJECT }} \ --repo-dir ${{ github.workspace }} \ --branch ${{ github.ref_name }} \ --loglevel INFO - name: "Upload crash dumps" uses: actions/upload-artifact@v3 if: failure() with: name: "crashdumps" path: ${{ github.workspace }}/.crashdumps/ retention-days: 3
